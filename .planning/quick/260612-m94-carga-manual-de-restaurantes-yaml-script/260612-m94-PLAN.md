<objective>
Crear una herramienta de operador para cargar/actualizar restaurantes desde un archivo YAML
simple, sin pasar por el frontend. Un solo comando valida el archivo, hace upsert de
restaurant + horarios + menú en Supabase (con service role, bypass RLS) y resincroniza el
assistant de Vapi. Incluye los datos reales del piloto Wonder y una guía en español para
un operador no programador. Cierra las fases 1-2 (web UI) como superseded por carga manual.

Purpose: El wizard web de onboarding (Fase 2) quedó con bugs (slug_taken) y es overhead para
un piloto de 1 restaurante. La carga manual por YAML es más rápida, robusta y mantenible para
el dueño del proyecto.

Output: lib/restaurant-yaml.ts (puro), scripts/cargar-restaurante.ts (CLI), wonder.yaml,
COMO-CARGAR.md, tests, y docs de planning actualizados.
