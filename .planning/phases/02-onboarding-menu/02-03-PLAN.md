## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | PATCH /menu-items/:id/availability on another tenant's item | mitigate | Two-layer: requireAuth + `.eq('restaurant_id', req.restaurantId)` on update; RLS as floor (Phase 1 policy on menu_items line 73 of 0001) |
| T-03-02 | Tampering | category_id from one tenant attached to item insert by another tenant | mitigate | POST /menu-items validates `category_id` belongs to `req.restaurantId` BEFORE insert (returns 400 invalid_category); RLS also blocks |
| T-03-03 | Tampering | Mass assignment (e.g., changing restaurant_id via PATCH body) | mitigate | Whitelist body fields explicitly in PATCH handler; never spread `req.body` |
| T-03-04 | Information Disclosure | Realtime channel leaks menu_items rows across tenants | mitigate | Plan 01 publication added; RLS on menu_items already filters by `app_metadata.restaurant_id` (Phase 1, 0001 line 73). Verified pattern per RESEARCH.md Pattern 3 |
| T-03-05 | Denial of Service | Nested write with malicious payload (1000 option_groups) | accept | MVP volume — no DoS protection. Rate limit added in Phase 6 (SEC-01). Acceptable per REQUIREMENTS scope. |
| T-03-06 | Tampering | Template loader overwriting existing menu | mitigate | Idempotency check returns 409 `template_already_loaded`; never destructive |
</threat_model>
